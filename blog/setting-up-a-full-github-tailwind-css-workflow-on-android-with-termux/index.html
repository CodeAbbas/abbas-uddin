Setting Up a Full GitHub + Tailwind CSS Workflow on Android with Termux

Abbas Uddin
23/10/2025 7 min read

Can you actually build modern websites entirely from an Android phone? Absolutely! Using powerful tools like Termux, SPCK Editor, and Node.js, you can manage a GitHub project and even run build processes like the Tailwind CSS CLI. It's a game-changer for coding on the go.

However, this setup isn't without its challenges. I recently spent hours troubleshooting a mysteriously broken Tailwind installation. This post documents the entire process we went through, including the crucial fix, so you can set up your mobile workflow smoothly.

The Toolkit

  • Termux: The core Linux environment for Android. Install `git` and `nodejs` within it using `pkg install git nodejs`.
  • SPCK Editor: A great code editor for Android with built-in Git support, making file management and commits easy.
  • Node.js/npm: Needed for installing and running Tailwind CSS.
  • A Local Server: Optional but recommended for previewing. `http-server` (via npm) is a good choice.

Step 1: Cloning and Editing with Git

Getting your project onto your phone is straightforward. Open Termux:

# Give Termux access to storage (run once)
termux-setup-storage

# Navigate to your preferred projects directory
cd /storage/shared/YourFolder

# Clone your repo
git clone https://github.com/YourUsername/YourProject.git
cd YourProject

You can now open this folder in SPCK Editor. Edit your files visually. When ready to save:

# Stage your changes
git add .

# Commit them
git commit -m "Made changes from Android"

# Push to GitHub (You might need to set up authentication first)
# For password/token auth:
git push origin main
# Enter username and Personal Access Token when prompted

So far, so good. But what about compiling Tailwind?

Step 2: Installing Tailwind CSS (The First Attempt)

My project initially used the Tailwind CDN. To switch to the CLI, I ran:

# Make sure you have a package.json (run if needed)
npm init -y

# Install Tailwind
npm install -D tailwindcss

I configured `tailwind.config.js` to scan my HTML and JS files:

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./index.html",
    "./script.js" // Include JS if classes are added dynamically
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

Added the directives to the top of my input CSS file (`style.css`):

@tailwind base;
@tailwind components;
@tailwind utilities;

/* My custom styles below... */
body { /* ... */ }

And ran the build command:

npx tailwindcss -i ./style.css -o ./output.css --watch

Finally, I updated `index.html` to remove the CDN script and link to `output.css`.

Step 3: The Failure and Troubleshooting

The build command ran without errors, but the website looked completely unstyled. The `output.css` file was generated, but it was tiny and missing all the Tailwind utility classes defined in my HTML and JS. Only my custom styles from `style.css` were present.

My first suspect was browser caching. I tried everything: hard refreshes, incognito tabs, clearing Chrome's site data for localhost, even clearing the entire app cache. I also tried a different local server (`http-server -c-1`) to disable server-side caching. None of it worked. The problem was deeper.

The Real Culprit: Tailwind v4 vs v3

The breakthrough came when I examined the generated `output.css` file. The first line revealed the issue:

/*! tailwindcss v4.1.15 | MIT License | https://tailwindcss.com */

Running `npm install -D tailwindcss` had installed the latest **Tailwind v4 beta**! My `tailwind.config.js` file, however, was written using the **v3 syntax** (with the `content: []` array).

Crucially, Tailwind v4 ignores the `content` array. It uses a different mechanism (often relying on framework integration or CSS `@import "tailwindcss" source(...)`) to find your template files. Because my config was v3 style, the v4 engine wasn't scanning `index.html` or `script.js` at all. This silent failure meant no utility classes were generated.

The Correct Fix: Installing Tailwind v3

The solution was to explicitly install the stable v3 version that matched my configuration:

# 1. Uninstall the v4 packages (including the separate CLI)
npm uninstall tailwindcss @tailwindcss/cli

# 2. Install the stable v3 package (CLI is included)
npm install -D tailwindcss@3

After this, running the build command again worked perfectly:

npx tailwindcss -i ./style.css -o ./output.css --watch

The v3 engine correctly read the `tailwind.config.js`, scanned the files listed in `content`, and generated a complete `output.css` file. A final browser refresh (after ensuring the cache was truly cleared) brought the correct styles to life.

Conclusion

Coding on Android with Termux is incredibly capable, allowing a full Git and Node.js workflow. However, version mismatches, especially with rapidly evolving tools like Tailwind CSS, can cause baffling issues. If your build seems broken despite correct configuration and cleared caches, always double-check your installed package versions against the documentation for the syntax you're using. With the right versions aligned, your mobile coding setup can be surprisingly productive!
