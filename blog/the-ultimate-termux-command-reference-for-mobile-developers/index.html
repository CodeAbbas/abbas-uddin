The Ultimate Termux Command Reference for Mobile Developers

Abbas Uddin
03/12/2025 6 min read

For developers who value portability and efficiency, Termux is an indispensable tool. It transforms your Android device into a powerful Linux environment, allowing you to code, manage files, and version control your projects from anywhere.

Whether you are fixing a bug on the train or setting up a full development environment on your tablet, knowing the right commands is key. Below is a comprehensive reference guide to the essential Termux commands you need to know.

I. Installation & Environment Setup

Before you start coding, you need to ensure your environment is stable and up to date. Run these commands immediately after installing Termux.

  • pkg update
    Updates the local package list. Essential before any install or upgrade.
  • pkg upgrade
    Upgrades all installed packages to keep your system secure and stable.
  • pkg install <package>
    Installs a new package (e.g., pkg install nodejs, pkg install git).
  • termux-setup-storage
    Grants Termux access to shared Android storage. This is crucial for accessing files in /sdcard or your downloads folder.
  • termux-change-repo
    Allows you to change the package repository mirror if download speeds are slow.

II. File Management & Navigation

Navigating a command-line interface requires muscle memory. These commands allow you to move through directories and manipulate files efficiently.

Command Function
pwdPrint Working Directory. Shows your current location.
ls -alList all files in long format (shows permissions, size, hidden files).
cd <path>Change Directory (e.g., cd ~/storage/downloads/).
mkdir -p <path>Make Directory. The -p flag creates parent directories if they don't exist.
mv <src> <dest>Move or Rename files/directories.
cp -r <src> <dest>Copy files or directories recursively.
rm -r <file/dir>Remove files or directories. Caution: Deletion is permanent.
nano <file>Opens the Nano text editor for quick file modifications.
cat <file>Concatenate (print) file contents to the screen.

III. Version Control (Git)

For any developer, Git is non-negotiable. Termux handles Git workflows flawlessly, allowing you to sync with GitHub or GitLab.

# Start a new repository
git init

# Clone an existing project
git clone https://github.com/username/repo.git

# Stage all changes
git add .

# Save changes with a message
git commit -m "My update message"

# Upload to remote server
git push origin main

# Update local project
git pull origin main

# Check file status
git status

IV. Development (Node.js/JavaScript)

As a Front-End developer, your stack likely revolves around JavaScript. Here is how to manage your Node environment in Termux.

  • Run JS files: node file.js
  • Initialize Project: npm init -y (creates a default package.json)
  • Install Dependencies: npm install express (or any other package)
  • Start Application: npm start (runs the script defined in package.json)
  • Scaffold Projects: npx create-next-app (useful for Next.js setups)

V. Networking & Utilities

These utilities help you diagnose connection issues or integrate Termux with the rest of your Android system.

ping <host>

Tests network connectivity (e.g., ping google.com).

ifconfig

Displays network interface configurations, including your IP address.

sshd

Starts the Secure Shell Daemon, allowing you to SSH into your phone from a computer.

termux-open <file>

Opens a file using the default Android app (e.g., open an HTML file in Chrome).

Conclusion

Termux is more than just a terminal emulator; it is a bridge that brings desktop-grade development tools to your mobile device. By mastering these commands, you can maintain productivity regardless of your location.
